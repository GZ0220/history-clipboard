# 质量加固与诊断说明

## 本轮目标

本轮重点不是增加复杂功能，而是提升日常使用的稳定性、流畅度和可排查性。主程序仍保持本地、离线、便携，OCR 仍作为可选组件分发。

## 流畅度优化

- Windows 环境优先使用 `GetClipboardSequenceNumber` 判断剪贴板是否变化。
- 剪贴板没有变化时，软件不会反复读取图片剪贴板，减少长期运行负担。
- 图片读取、图片哈希、大小判断继续放在后台线程。
- 置顶、取消置顶、复制更新时间、备注和 OCR 保存采用延迟保存，减少连续点击时的磁盘写入。
- 新增、删除、批量删除、清理等关键数据变更仍立即保存。
- 退出、切换保存位置、最小化到托盘前会强制保存待写入数据。
- 历史列表采用虚拟列表，只渲染当前屏幕附近的卡片控件，历史记录很多时也不会让 Tkinter 一次性重绘全部卡片。

## 诊断与压测

- “数据管理”中新增“状态诊断”入口。
- 诊断窗口显示版本、主程序目录、数据目录、配置文件、历史数量、OCR 状态、剪贴板序号检测状态和最近性能事件。
- 可导出 `历史粘贴板诊断_YYYYMMDD_HHMMSS.txt`。
- 开发调试时可设置环境变量 `CLIPBOARD_HISTORY_PERF=1` 查看性能日志。
- 可用 `python main.py --perf-seed 测试保存文件夹 500` 构造测试数据，用于验证大量记录下的搜索、置顶、复制和删除流畅度。

## 日常控制

- 默认启用全局快捷键 `Ctrl+Alt+V`，只用于快速呼出/聚焦窗口，不会自动粘贴内容。
- “设置”中可关闭或重新开启全局快捷键，并查看注册状态：可用、被占用或注册失败、当前系统不支持。
- “设置”入口放在顶部常用操作区，不放在“数据管理”折叠区里。
- “设置”弹窗集中管理全局快捷键、开机自启、敏感过滤、单张图片上限和最大记录数，避免顶部按钮平铺过多。
- “单张图片上限”只限制单次复制的某一张图片，不代表所有图片的总容量上限。
- 开机自启默认关闭，用户可手动开启。
- 暂停记录已整理为“记录状态”下拉菜单，支持继续记录、手动暂停、暂停 5 分钟、暂停 30 分钟。
- 敏感文字过滤默认关闭；开启后会忽略明显密码、验证码、token 类短文本。

## 窗口恢复

- 从任务栏或托盘恢复窗口时，只恢复当前虚拟列表窗口，不重新创建全部历史卡片。
- 普通任务栏最小化时会释放当前渲染卡片到卡片池；从任务栏恢复时延迟渲染当前视口，避免窗口刚出现就重绘大量控件。
- 托盘图标保持常驻，只有完全退出时才停止，减少每次恢复窗口时的额外阻塞。
- 恢复窗口不会重新查询或重建完整历史列表，减少黑框、空白和二次加载感。

## 数据安全

- 恢复备份前会提示确认。
- 恢复备份前会自动创建一份当前数据备份，避免误覆盖后无法找回。
- 备份恢复不改变历史数据结构，继续使用 `history_index.json`、`images/` 和 `thumbs/`。
