# 打包与分发说明

## 目标效果

用户双击 `历史粘贴板.exe` 即可运行。对方电脑不需要提前安装 Python，也不需要打开命令行。

## 推荐分发方式

第一版推荐使用 PyInstaller 的文件夹模式，而不是单文件模式。

打包完成后发送整个文件夹：

```text
dist/历史粘贴板/
```

这个文件夹里会包含：

- `历史粘贴板.exe`
- Python 运行时文件
- Pillow、Tkinter 等依赖文件
- 应用图标资源

只发送单独的 exe 可能无法运行，因为依赖文件也在同一个文件夹中。

当前已生成的发布目录：

```text
dist/历史粘贴板/
```

OCR 现在作为可选组件单独生成：

- `dist/历史粘贴板/`：默认轻量主程序，不包含 OCR 依赖。
- `dist/OCR组件/ocr_runtime/`：可选 OCR 组件，包含 RapidOCR、ONNX Runtime 和离线识别模型。

如果用户需要图片文字识别，可以把整个 `ocr_runtime` 文件夹复制到 `历史粘贴板.exe` 所在目录旁边；也可以把整个 `OCR组件` 文件夹放到 `历史粘贴板.exe` 同级目录。不要只复制里面的单个 exe。

## 本机打包步骤

1. 安装打包工具：

```powershell
python -m pip install -r requirements-dev.txt
```

2. 运行打包脚本：

```powershell
build_exe.bat
```

3. 打包成功后，打开：

```text
dist/历史粘贴板/
```

4. 双击 `历史粘贴板.exe` 测试。

也可以先运行自检：

```powershell
dist\历史粘贴板\历史粘贴板.exe --self-test
```

如果命令正常退出，说明 exe 至少可以加载程序入口和依赖。

主程序会使用 `assets/app_icon.ico` 作为窗口图标和 exe 图标。若图标没有更新，通常是旧版 exe 正在运行导致打包目录被占用，关闭旧进程后重新打包即可。

也可以单独测试 OCR 组件：

```powershell
dist\OCR组件\ocr_runtime\ocr_worker.exe 测试图片.png
```

## 便携配置说明

exe 版本会把 `clipboard_history_config.json` 保存在 exe 同目录。第一次运行时，软件仍会让用户选择历史数据保存文件夹。

如果把软件发给别人，建议只发送干净的 `dist/历史粘贴板/` 文件夹，不要把自己的历史数据文件夹一起发出去。

如果对方不需要 OCR，只发送 `dist/历史粘贴板/` 即可；如果对方需要 OCR，再额外发送 `dist/OCR组件/` 或其中的 `ocr_runtime/`。
