# OCR 组件与批量管理说明

## 分发结构

- `dist/历史粘贴板/`：默认轻量主程序，双击 `历史粘贴板.exe` 即可使用基础剪贴板历史功能。
- `dist/OCR组件/ocr_runtime/`：可选 OCR 组件，需要图片文字识别时，可以把整个 `ocr_runtime` 文件夹复制到 `历史粘贴板.exe` 所在目录旁边，也可以把整个 `OCR组件` 文件夹放到 `历史粘贴板.exe` 同级目录。

## OCR 工作方式

主程序不直接打包 RapidOCR 和 ONNX Runtime。点击“识别文字”时，主程序会查找旁边的 `ocr_runtime/ocr_worker.exe` 或 `OCR组件/ocr_runtime/ocr_worker.exe`，并把图片路径传给它识别。

如果未检测到组件，软件只显示安装提示，不影响文字、图片记录、搜索、备注、导出和删除等基础功能。

## 图片卡片展示

图片记录中的“备注”和“识别结果”分开展示：

- 备注：用户手动输入，用于解释图片用途或来源。
- 识别结果：OCR 自动识别得到的文字，可复制、保存并参与搜索。
- 默认隐藏卡片中的识别结果，只显示“已识别”标记；需要查看预览时可点击顶部“显示识别结果”。

隐藏识别结果只影响卡片显示，不影响搜索。搜索 OCR 文本时仍会命中对应图片卡片。

## 批量管理

每张历史卡片左侧提供勾选框。批量工具栏支持：

- 全选当前结果：只选择当前搜索和筛选后的可见记录。
- 取消选择：清空当前勾选。
- 批量复制：生成文字汇总；文字记录复制原文，图片记录复制图片路径、备注和 OCR 文本。
- 批量导出：生成带时间戳的导出文件夹，包含 `selected_index.json`、`selected_text.txt`、图片和缩略图副本。
- 批量删除：删除前二次确认，避免误删。
